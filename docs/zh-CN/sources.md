# 媒体源说明

本文档说明当前版本里媒体源的接入方式和注意事项。

## 设计思路

当前版本不是把“媒体源”做成完全独立的插件系统，而是在原有 `ani-cli` 的播放链路上，为不同站点补充各自的抓取逻辑。

中文站点接入时，部分站点选择、搜索入口和页面规则参考了 [MajoSissi/animeko-source](https://github.com/MajoSissi/animeko-source) 中公开可见的订阅源定义。

一条典型链路通常包括：

1. 搜索番剧
2. 获取详情页
3. 解析分集列表
4. 打开播放页
5. 提取最终可播放链接
6. 交给播放器播放

## 当前涉及的源

### `allanime`

原项目默认源。

适合原始 `ani-cli` 使用方式。

### `senfun`

中文站点适配的实验性实现之一。

它的特点是：

- 搜索页与详情页结构比较直接
- 可通过站点接口拿到播放数据
- 在部分播放器和网络环境下可能仍然不稳定

### `omofun111`

当前版本重点适配的中文站点之一。

它的特点是：

- 搜索结果可从站点搜索页提取
- 分集列表可从详情页中解析出 `/vod/play/.../nid/...` 路径
- 播放页中可进一步提取最终 `m3u8` 或直链地址

## 订阅源参考

当前版本没有直接把 `animeko-source` 当成运行时依赖导入，而是把其中公开可见的站点规则作为适配参考。

可理解为：

- `ani-cli` 提供命令行交互和播放主流程
- `animeko-source` 提供了中文站点规则参考
- `kanfan-cli` 在此基础上实现面向 shell 脚本的适配逻辑

## 为什么需要单独适配

不同站点的页面结构、参数命名、播放页脚本、直链字段和请求头要求都不一样，所以不能简单地只替换一个 URL。

当前版本为中文站点补的主要是：

- 搜索结果选择器
- 分集列表提取逻辑
- 播放页最终链接提取逻辑
- 与播放器兼容的 `Referer` 处理

## 稳定性说明

这类站点适配天然会受以下因素影响：

- 站点改版
- 域名切换
- 播放页脚本变化
- CDN 与防盗链策略变化
- 搜索页返回结构变化

因此当前版本更适合作为“实验性适配”而不是永久稳定接口。

## 后续扩展建议

如果后面继续增加中文站点，建议保持以下思路：

- 每个站点单独维护搜索、分集、播放提取逻辑
- 先验证 `ANI_CLI_PLAYER=debug` 是否能拿到最终链接
- 再处理播放器兼容、请求头和网络出口问题
